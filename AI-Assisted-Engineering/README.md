## CLI Agents (Агенты в терминале)

CLI-агенты — это утилиты, которые индексируют проект, читают/правят файлы, запускают тесты и выполняют команды через системные вызовы, оставаясь внутри терминала.

> **Глубокий разбор:** реальные кейсы установки, обхода ограничений и сравнение Claude Code vs OpenClaude вынесены в отдельный каталог — [Claude Code & OpenClaude](Claude-Code-and-OpenClaude/README.md).

### Сравнение инструментов

| Критерий | Claude Code | Cline (VS Code) | Copilot CLI |
| :--- | :--- | :--- | :--- |
| **Среда** | Терминал (CLI) | Расширение IDE | Терминал (CLI) |
| **Модель** | Claude (Anthropic) | Любая через API/роутер | GPT (OpenAI) |
| **Вызов инструментов** | Нативные tool calls (Read, Bash, Edit) | Tool calls внутри IDE | Подсказки команд shell |
| **Контекст проекта** | Индексация + `grep`/glob по репо | Открытые файлы + workspace | Текущая команда/история |

### Ключевые ограничения

- **Контекстное окно** — большой репозиторий не помещается целиком; агент опирается на поиск (`grep`, glob), а не на полное чтение.
- **Оверхед на сбор контекста** — частые вызовы поиска и «оркестровка» подагентов увеличивают латентность одной итерации.
- **Недетерминизм** — один и тот же промпт может дать разные правки; для прод-пайплайнов нужна верификация (тесты, ревью).

## LLM API & Routing (Маршрутизация запросов)

Чтобы CLI-агент работал стабильно и предсказуемо по стоимости, запросы маршрутизируют через шлюз-агрегатор.

### OpenRouter как единая точка входа

- **Агрегация провайдеров** — один API-ключ и единый эндпоинт дают доступ к моделям Anthropic, OpenAI, Google и др.
- **Переопределение эндпоинта** — многие CLI читают переменные окружения, что позволяет завернуть трафик на шлюз:

```bash
export ANTHROPIC_API_KEY="ключ_openrouter"
export ANTHROPIC_BASE_URL="https://openrouter.ai/api/v1"
```

### Ротация ключей и балансировка

- **Key Rotation** — пул ключей с переключением при достижении rate limit; снижает риск блокировки одного ключа.
- **Cost/Token Balancing** — дешёвые модели на рутину (генерация документации, простые правки), дорогие — на сложные рассуждения и архитектуру.
- **Фолбэк** — при недоступности или таймауте основной модели запрос автоматически уходит на резервную.

## Context Optimization (Оптимизация контекста)

Главная статья расходов и латентности — повторная передача одного и того же контекста на каждый запрос.

### Prompt Caching на стороне провайдера

- **Механизм** — стабильная часть промпта (системные инструкции, большие файлы, документация) кэшируется провайдером; повторные запросы переиспользуют кэш по сниженной цене и с меньшей задержкой.
- **TTL** — кэш живёт ограниченное время (обычно минуты); длинные паузы между запросами «остужают» кэш и приводят к повторной полной оплате.

### Структурирование рабочего пространства

- **Стабильное в начало** — то, что не меняется между запросами, размещать первым (для попадания в префиксный кэш).
- **Изменяемое в конец** — конкретный вопрос/дифф ставить последним.

### Сокращение входных токенов через ignore-конфигурации

Файлы вроде `.claudeignore` исключают мусор из индексации и не дают агенту тратить токены на лишнее:

```gitignore
node_modules/
dist/
*.log
.terraform/
*.tfstate
```

## Prompt Engineering for IaC (Промпты под инфраструктуру)

Генерация инфраструктурного кода требует жёстких системных промптов: цена галлюцинации — сломанный деплой или утечка прав.

### Kubernetes-манифесты без галлюцинаций

- **Фиксировать `apiVersion`** — модель склонна выдумывать устаревшие версии API; их надо явно задавать в промпте.
- **Запрет на лишние поля** — требовать только валидные ключи спецификации, без «правдоподобных» несуществующих параметров.

### Идемпотентные Ansible-плейбуки

- **Декларативность** — требовать описание *желаемого состояния* (`state: present`), а не последовательность `shell`-команд.
- **Идемпотентность** — повторный прогон не должен менять систему; модулю `command`/`shell` всегда добавлять `creates`/`changed_when`.

### Валидный HCL (Terraform)

- **Версии провайдеров** — требовать блок `required_providers` с пиннингом версий, чтобы код не «плыл».
- **Без хардкода секретов** — переменные и `*.tfvars`/secret-бэкенды вместо значений в коде.

> **Общий приём:** в системном промпте требовать самопроверку синтаксиса и прогон линтеров (`kubeval`/`kubeconform`, `ansible-lint`, `terraform validate`) до выдачи финального ответа.
