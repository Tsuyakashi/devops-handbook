# Таска 3. AWS + SSH

## Цель
Подготовить окружение в AWS, настроить пользователей и права доступа, а также организовать хранение конфигураций и документации.

---

## Задача 1. Репозиторий и документация

1. Создать **приватный репозиторий на GitHub** для дальнейшей работы  
   (для сохранения всех будущих конфигов nginx, docker и т.д.).
2. Создать аккаунт в **Notion**, **NotebookLM** или аналог для ведения документации по таскам.  
   Документацию по всем шагам задачи вести **обязательно**.

---

## Задача 2. AWS и бюджет

1. В AWS‑аккаунте настроить **алерты на превышение недельного бюджета 7$**,  
   чтобы уведомления приходили на почту или телефон.
2. Создать **инстанс** в AWS **минимально возможного тира**.

---

## Задача 3. Пользователи и доступ

1. На инстансе создать пользователей:  
   `user1`, `user2`, `user3`, `user4`, `user5`.
2. Настроить доступ:
   - **user1** должен заходить **только по SSH‑ключу**.  
   - **user2** должен **каждый раз вводить пароль при входе по SSH**.
3. Настроить права:
   - **user1** должен иметь **доступ к sudo**.  
   - **user2** **не должен иметь** доступ к sudo.

---

## Задача 4. Файл и права доступа

1. Под пользователем **user1** создать документ в директории `/opt`.
2. Настроить права доступа так, чтобы **у всех пяти пользователей были разные комбинации прав**  
   (ни одна комбинация не должна повторяться).

---

## Задача 5. Интерпретатор по умолчанию

1. Для **user3** настроить **интерпретатор по умолчанию `sh`**.  
2. Если по умолчанию уже используется `sh`, изменить его на **`bash`**.
