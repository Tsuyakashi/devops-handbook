# Таска 4. Nginx proxy + upstream + regex

## Цель
Настроить nginx как обратный прокси, балансировщик и сервер статики, а также включить HTTPS и обработку разных маршрутов.

---

## Базовый nginx и страницы

- Поднять nginx.
- Создать дефолтную страницу со следующим содержанием:
  - «Перейти на другую страничку» — гиперссылка на страницу, которая обрабатывается этим же nginx: `host/secondpage`.
  - «Скачать музыку» — гиперссылка, при нажатии происходит скачивание музыкального файла: `host/music`.
  - «Получить информацию о PHP сервере» — гиперссылка с переходом на Apache с выводом информации о PHP: `host/info.php`.
  - «Получить респонс с другого сервера» — гиперссылка на сайт, который отдаётся не этим proxy, а другим сервером: `host/secondserver`.

---

## Обработка изображений и regex

- Настроить переход на `/image1` для изображений формата `.jpg`.
- Настроить переход на `/image2` для изображений формата `.png`.
- Для png-картинок при выдаче нужно поворачивать изображение на 180 градусов.

---

## HTTPS и DNS

- Зарегистрироваться на сайте noip.com (или аналогичном сервисе).
- Разобраться в типах DNS-записей и создать DNS-запись типа **A** для своего тестового сервера.
- Получить SSL-сертификат (например, Let's Encrypt, webroot и т.п.).
- В nginx добавить редирект со всех подключений на порту 80 на порт 443.

---

## Балансировка и проксирование

- Создать ещё два сервера в nginx.
- Настроить маршрут `/redblue` так, чтобы при обновлении страницы происходило чередование красной и синей страниц  
  (балансировка между двумя серверами).
- В логах вывести информацию, на какой из серверов был проксирован запрос клиента.
