# Task 4: Nginx Proxy + Upstream + Regex

## Описание проекта

Проект настраивает Nginx как обратный прокси-сервер, балансировщик нагрузки и сервер статики с поддержкой HTTPS, обработкой различных маршрутов и фильтрацией изображений.

## Функциональность

### Основные возможности

1. **Обратный прокси** - Nginx проксирует запросы к различным бэкенд-серверам
2. **Балансировка нагрузки** - Распределение запросов между красным и синим серверами
3. **HTTPS** - Автоматический редирект с HTTP на HTTPS, SSL-сертификаты от Let's Encrypt
4. **Обработка статики** - Раздача HTML, изображений и файлов
5. **Фильтрация изображений** - Поворот PNG изображений на 180 градусов через модуль image_filter

### Доступные эндпоинты

- `/` - Главная страница с навигацией
- `/secondpage` - Вторая страница
- `/music` - Скачивание музыкального файла
- `/info.php` - Информация о PHP сервере (проксируется на PHP контейнер)
- `/secondserver` - Проксирование на внешний сервер (Pacman)
- `/image1` - Отображение JPG изображения
- `/image2` - Отображение PNG изображения (повернуто на 180°)
- `/redblue` - Балансировка между красным и синим серверами
- `/doom`, `/keen`, `/scorch`, `/bomberman` - Дополнительные игровые серверы

## Требования

- Ubuntu/Debian система
- Docker и Docker Compose (для локального тестирования)
- Доступ с правами sudo
- Для KVM режима: libvirt, qemu
- Для AWS режима: AWS CLI, настроенные credentials

## Установка и запуск

### Режим KVM (по умолчанию)

Скрипт автоматически создаст или запустит виртуальную машину Ubuntu и настроит на ней все сервисы:

```bash
./init.sh
# или явно указать режим
./init.sh --mode KVM
```

**Требования:**
- Права sudo
- SSH ключ в `./keys/rsa.key`
- Настроенный KVM

### Режим AWS

Запуск на AWS EC2 инстансе:

```bash
./init.sh --mode AWS
```

**Требования:**
- Настроенный AWS CLI
- SSH ключ в `./keys/$KEY_PAIR_NAME.pem`
- Security Group с открытыми портами 22, 80, 443

### Локальное тестирование с Docker Compose

Для тестирования отдельных серверов (red, blue, php) можно использовать Docker Compose:

```bash
docker compose up -d
```

Это запустит:
- `red-server` на порту 8001
- `blue-server` на порту 8002
- `php-server` на порту 8008

## Что делает скрипт init.sh

1. **Установка пакетов** - Проверяет и устанавливает nginx, docker.io
2. **Запуск контейнеров** - Запускает Docker контейнеры для бэкенд-серверов:
   - Red server (порт 8001)
   - Blue server (порт 8002)
   - PHP server (порт 8008)
   - Pacman server (порт 8003)
   - Дополнительные игровые серверы (порты 8004-8007)
3. **Установка модуля image_filter** - Добавляет официальный репозиторий Nginx и устанавливает модуль для обработки изображений
4. **Настройка Nginx** - Создает конфигурацию с:
   - Редиректом HTTP → HTTPS
   - SSL сертификатами (Let's Encrypt)
   - Upstream блоками для балансировки
   - Location блоками для всех маршрутов
5. **Получение SSL сертификата** - Использует Certbot для получения сертификата
6. **Размещение файлов** - Копирует статические файлы в `/opt/dkt/`

## Конфигурация Nginx

Основная конфигурация находится в `/etc/nginx/conf.d/dkt.conf` и включает:

### Upstream блоки

- `php_server` - PHP сервер на localhost:8008
- `secondserver` - Внешний сервер на localhost:8003
- `redblue_servers` - Балансировка с алгоритмом `least_conn` между localhost:8001 и localhost:8002

### Особенности

- Автоматический редирект HTTP → HTTPS
- HTTP/2 включен
- Логирование балансировки в `/var/log/nginx/redblue_access.log`
- Обработка изображений через модуль `ngx_http_image_filter_module`

## DNS и SSL

Проект использует домен `trainee.servebeer.com` (настраивается в скрипте).

**Требования:**
- DNS A-запись, указывающая на IP сервера
- Certbot для получения SSL сертификата от Let's Encrypt
