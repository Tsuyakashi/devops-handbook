# Описание скрипта AWS и Linux пользователей

## 1. Создание EC2 инстанса (опция `--aws`)

- Получение default VPC ID
- Создание (если нет) Security Group с именем из переменной `SECURITY_GROUP_NAME`
- Разрешение входящих SSH (TCP порт 22) для группы безопасности
- Получение Subnet ID в default AZ
- Проверка существования ключа доступа AWS `KEY_PAIR_NAME`, создание при отсутствии
- Запуск EC2 инстанса с заданным AMI (`AMI_ID`), типом, ключом и группой безопасности
- Ожидание запуска инстанса и получение публичного IP
- Ожидание доступности SSH (порта 22)
- Копирование скрипта `init.sh` на инстанс и запуск с sudo

## 2. Управление пользователями и доступом

- Создание пользователей: `user1`, `user2`, `user3`, `user4`, `user5` с одним паролем `root123`
- Настройка SSH доступа:
  - `user1`: доступ только по SSH ключу (пароль отключен)
  - `user2`: доступ только по паролю (ключ отключен)
- Расположение SSH авторизованных ключей для user1 копируется из текущего пользователя `ubuntu`
- Изменение конфигурации sshd (`/etc/ssh/sshd_config.d/60-cloudimg-settings.conf`) для указанных пользователей
- Перезапуск ssh сервиса
- Добавление `user1` в группу sudo
- Добавление для `user1` права выполнять sudo через файл `/etc/sudoers.d/90-cloud-init-users`

## 3. Права доступа к файлу /opt/myfile.txt

- Установка ACL прав:
  - `user1`: чтение и запись (`rw-`)
  - `user2`: только чтение (`r--`)
  - `user3`: только запись (`-w-`)
  - `user4`: чтение и выполнение (`r-x`)
  - `user5`: нет доступа (`---`)

## 4. Настройка интерпретатора оболочки для user3

- Проверка текущего shell пользователя user3
- Если установлен `/bin/sh`, сменить на `/bin/bash`, иначе наоборот

---

## Переменные, используемые в скрипте

| Переменная           | Описание                              | Значение по умолчанию                  |
|----------------------|-------------------------------------|--------------------------------------|
| `AMI_ID`             | ID AMI для запуска EC2 инстанса      | `"ami-004e960cde33f9146"`            |
| `EC_COUNT`            | Количество инстансов                 | `1`                                  |
| `INSTANCE_TYPE`       | Тип инстанса                        | `"t2.micro"`                         |
| `KEY_PAIR_NAME`       | Имя ключа EC2                       | `"amzn-key"`                         |
| `SECURITY_GROUP_NAME` | Имя Security Group                  | `"CLI-security-group"`                |

---

## Требования

- Установленный и настроенный AWS CLI
- Наличие файла `init.sh` для инициализации EC2 инстанса
- Права sudo для выполнения изменений в системе
- Установлен пакет `acl` для установки ACL прав

---

## Запуск

- Чтобы запустить создание AWS инстанса и начальную инициализацию:

```bash
./init.sh --aws
```


- Для настройки пользователей и прав доступа просто запустить скрипт без параметров:

```bash
./init.sh 
```

---

## Ошибки и проверки

- Если AWS CLI не установлен — выводится сообщение и выход
- Несоответствие поддерживаемого количества инстансов вызывает ошибку
- Все критичные операции имеют проверку возвратного значения

---

# Краткая инструкция выполнения

1. Запустите скрипт с `--aws` для развертывания EC2 (подождите доступности SSH)
2. SSH под root или ubuntu и выполните дополнительную настройку (если требуется)
3. Запустите скрипт для создания пользователей и установки прав
