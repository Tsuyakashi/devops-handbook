# Скрипт автоматизации AWS EC2 и настройки пользователей Linux

## Описание скрипта

Скрипт выполняет две основные задачи:
- Развёртывание EC2 инстанса в AWS с базовой настройкой (при запуске с опцией `--aws`)
- Конфигурация пользователей и прав доступа на Linux хосте (без параметров)

---

## 1. Развёртывание EC2 инстансов (опция `--aws`)

### Основные шаги

- Получение ID default VPC
- Создание Security Group, если она отсутствует, с разрешением входящего TCP порта 22 (SSH)
- Получение Subnet ID для default availability zone
- Проверка наличия EC2 Key Pair, создание при отсутствии и сохранение в каталоге `./keys`
- Запуск EC2 инстансов (количество задано переменной `EC_COUNT`)
- Ожидание полной готовности инстанса (статус `running`)
- Получение публичного IP-адреса инстанса
- Ожидание доступности SSH-порта (22) на инстансе
- Копирование скрипта `init.sh` на инстанс и его удалённый запуск с sudo

### Переменные конфигурации

| Переменная           | Описание                         | Значение по умолчанию          |
|----------------------|---------------------------------|------------------------------|
| `AMI_ID`             | Идентификатор AMI AWS           | `"ami-004e960cde33f9146"`    |
| `EC_COUNT`            | Количество создаваемых инстансов| `1`                          |
| `INSTANCE_TYPE`       | Тип инстанса                    | `"t2.micro"`                 |
| `KEY_PAIR_NAME`       | Имя ключа для SSH доступа       | `"some-key-pair"`            |
| `SECURITY_GROUP_NAME` | Имя Security Group              | `"CLI-security-group"`       |

### Особенности

- Поддерживается создание нескольких инстансов (`EC_COUNT`)
- Для каждого запускается `init.sh` после копирования
- Если AWS CLI отсутствует, скрипт прерывается с сообщением

---

## 2. Конфигурация пользователей Linux (функция `userConfigure`)

### Создаем пользователей

- Пользователи: `user1`, `user2`, `user3`, `user4`, `user5` с паролем `root123`

### Настройка SSH доступа

- `user1`: только доступ по SSH ключу (пароль отключен)
- `user2`: только доступ по паролю (ключ отключен)
- Для `user1` копируются authorized_keys из пользователя `ubuntu`
- Конфигурация SSH редактируется через `/etc/ssh/sshd_config.d/60-cloudimg-settings.conf` с директивами `Match User`

### Служба SSH

- После изменений перезапускается `ssh` (`systemctl restart ssh`)

### Права sudo

- Пользователь `user1` добавляется в группу `sudo`
- В файл `/etc/sudoers.d/90-cloud-init-users` добавляются права на выполнение sudo для `user1`

### Работа с файлом и правами доступа

- Создается файл `/opt/myfile.txt`
- Устанавливаются ACL-права с разными комбинациями на каждого пользователя:

  | Пользователь | Права          |
  |--------------|----------------|
  | user1        | чтение и запись (`rw-`) |
  | user2        | только чтение (`r--`)   |
  | user3        | только запись (`-w-`)  |
  | user4        | чтение и выполнение (`r-x`) |
  | user5        | без доступа (`---`)     |

- Пакет `acl` устанавливается при необходимости

### Настройка оболочки пользователя `user3`

- Если текущий shell — `/bin/sh`, меняется на `/bin/bash`
- Если отличается — меняется на `/bin/sh`

---

## Запуск скрипта
- Для создания и настройки AWS инстансов:
```bash
./init.sh --aws
```
- Для настройки пользователей, прав SSH и ACL:
```bash
./init.sh 
```
---

## Ошибки и проверки

- Если AWS CLI не установлен — выводится сообщение и выход
- Несоответствие поддерживаемого количества инстансов вызывает ошибку
- Все критичные операции имеют проверку возвратного значения

---

# Краткая инструкция выполнения

1. Запустите скрипт с `--aws` для развертывания EC2 (подождите доступности SSH)
2. SSH под root или ubuntu и выполните дополнительную настройку (если требуется)
3. Запустите скрипт для создания пользователей и установки прав
