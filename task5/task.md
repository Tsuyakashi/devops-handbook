## Таска 5. Теория по промпт‑инжинирингу и bash

Цель: потренироваться в работе с bash, логами Nginx и применении LLM для анализа логов, а также закрепить базовую теорию по промпт‑инжинирингу.

Раздел 1. Nginx, bash и Python

Задача 1.1. Вывод загрузки CPU

- Вывести загрузку CPU на страничку Nginx в реальном времени с автоматическим обновлением.

Задача 1.2. Работа с логами Nginx

- Создать демон‑процесс, который раз в 5 секунд пишет логи Nginx в файл 1. При попытке убить процесс он должен перезапускаться.
- Когда файл 1 достигнет размера больше 300 КБ, он должен очищаться.
- В файл 2 записывать лог об удачных очистках файла 1: дата и время очистки, а также количество удалённых записей.
- В файл 3 складывать логи с кодами 5xx (пятисотые ошибки).
- В файл 4 складывать логи с кодами 4xx (четырёхсотые ошибки).
- Логи Nginx можно брать любые, до которых получится дотянуться. Для парсинга логов использовать sed и awk.

Задача 1.3. Использование LLM при анализе логов

- изучить теорию про промпт-инжинирнг
- Вводные: есть Python‑скрипт, который использует LLM для анализа логов Nginx. Скрипт принимает файл с логами и отдельный текстовый файл с инструкциями.
- Настроить окружение: установить все зависимости через pip install и использовать Gemini c google api key или локальную LLM, например ollama.
- Сформулировать промпт для анализа логов Nginx, который:
    - Находит запросы с кодом ответа 400.
    - Группирует эти запросы по IP‑адресам клиентов.
    - Подсчитывает количество запросов с кодом 400 для каждого IP.
    - Определяет наиболее частые endpoints, которые приводят к ошибкам.
    - Даёт краткое описание подозрительных закономерностей или аномалий (нужно формально описать, что считается аномалией).
- В каждом файле инструкций обязательно оставить плейсхолдер {logs}, куда будет подставляться содержимое файла с логами.
- Пример запуска скрипта:
    
    ./analyze_logs.py --logfile sample_nginx.log --promptfile instructions1.txt --temperature 0.2 --top_p 0.9 --seed 123
    
- Оценка: важна корректная работа скрипта с несколькими файлами инструкций, демонстрация экспериментов с разными формулировками задач и наблюдения за изменением результатов при разных настройках.

Раздел 2. Работа с процессами, дисками и файловой системой

Задача 2.1. Процессы

- Просмотреть список запущенных процессов.
- Создать и убить зомби‑процесс.
- Объяснить вывод команды ps auxft.
- Объяснить, какой процесс работает под PID 1.
- Показать дочерние процессы процесса с PID 1.
- Понять и уметь объяснить, что такое RSS, VSZ, NI, PRI, STAT.
- Определить, какой процесс используется текущим терминалом.

Задача 2.2. Открытые файлы в /var/log

- Определить процессы, которые имеют открытые файлы в директории /var/log.
- Объяснить вывод использованной команды (или команд).

Задача 2.3. Диски и inodes

- Показать, сколько места занято на дисках.
- Показать, сколько места на диске занимает директория /var/log/.
- Аналогично показать, сколько занято inodes на дисках и в директории /var/log/.
